# DB Env 경고 중복 억제 정책

`lib/db-env.ts`의 URL 충돌 경고 로그는 같은 프로세스에서 같은 시그니처에 대해 한 번만 출력됩니다.

## 목적

- `next dev`/QA 실행 중 동일 경고가 반복 출력되어 실제 이슈 탐지가 어려워지는 문제를 줄입니다.
- 환경값이 실제로 바뀐 경우에는 다시 경고를 출력해 변화는 놓치지 않습니다.

## 동작 규칙

1. 충돌 경고 시그니처는 `conflictMessages + selected keys` 조합으로 계산합니다.
2. 현재 시그니처가 직전 시그니처와 같으면 경고를 생략합니다.
3. 시그니처가 달라지면 경고를 다시 출력합니다.
4. 상태는 `globalThis.__WB_DB_ENV_WARN_STATE__`에 저장해 모듈 중복 로드 상황에서도 중복 경고를 최대한 줄입니다.

## QA 검증

```bash
npm run qa:db-env:warn-dedupe
```

검증 항목:

- 동일 충돌 시그니처 재검증 시 경고 1회 유지
- 충돌 시그니처 변경 시 경고 재출력
